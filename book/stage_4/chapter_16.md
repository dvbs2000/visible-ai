# 第 16 章 · 提示词工程：别误会，这是一场心理学构型仪式

![第 16 章 · 提示词工程：别误会，这是一场心理学构型仪式 · 四格漫画](../../images/sheets/chapter_16.jpg)

> ### 🎯 先别往下翻 · 这一章要破的题
>
> **🔥 痛点**：网上全是"ChatGPT 神级咒语大全"。写提示词，真得背口诀、对 AI 说"谢谢""求求你"吗？
> **🤔 换你来**：同一个问题，加一句"你是资深律师"，回答立刻变专业——这是 AI"入戏"了吗？
> **🧱 笨办法会撞墙**：把 prompt 当成"下给 AI 的命令"、靠揣摩它的"心情"——这样你只会**滑向收集咒语、迷信话术**，而且换个模型就失灵。
> 真相是：prompt 不是命令，是给"文字接龙"设的**续写条件**。往下看怎么在高维空间里"画个圈"。👇


第三阶段，元元带小满把一个 LLM 从无到有怎么炼成的全程围观完了。第四阶段第一章——**神兵出鞘第一招**，写提示词。

小满一上来就抛出个真实困惑：「网上那些'ChatGPT 神级咒语大全'……我得背口诀吗？写提示词真要对它说'谢谢''求求你'吗？」

元元"啪"地把一沓打印出来的"咒语大全"扔进垃圾桶：「**先把这迷信破了！**写提示词不是求神拜佛、不是揣摩 AI 的心情——它是一场**'心理学构型仪式'**：在高维空间里**画个圈**，给模型**套上一张性格面具**，强行把它拽进你想要的那片知识群落。今天我用第 8 章的星空给你讲明白（￣▽￣）ノ」

---

## 第 1 节　你打的每个字，都在给 AI 划"续写区"

元元先亮底牌：「第 12 章你已经知道，大模型唯一会做的事是**接着前文往下接龙**——每生成一个 token 前，都要对词表里几万个候选算一遍概率（第 14 章）。那 prompt 是什么？」

> **直觉印象**：prompt 是下给 AI 的命令 → 措辞够"讨喜"它就愿意好好干。
> **真实机制**：prompt 是续写的**条件** → 每个字都把后文分布**推向某片训练数据区**。

「按第一种理解，」元元说，「提示工程就成了揣摩 AI'心情'、收集咒语的迷信。但真相是——**提示工程 = 用文字给模型'圈地'**：把它的续写范围，从'全网平均'收窄到你想要的那片文本区域。」

他搬出第 8 章的**词向量星空**打比方：

> 🌌 **高维空间画圈 + 套性格面具**
> 你写「假装你是一位**资深合同律师**」，不是模型"入戏"这种魔法。训练数据里，"资深律师"这个词周围**聚集着海量法律文本**：合同条款、法律意见书、风险提示。这句话一出，后续 token 的概率分布就**对齐到那片区域**——措辞变严谨、主动逐条找风险、引用条款编号。
>
> **角色词，就是给模型套的一张"性格面具"**；戴上律师面具，它就被你拽进了星空里"法律"那片群落。角色扮演有效，原理就这么朴素。

元元现场演了个"圈地机"给小满看。同一个问题「解释一下量子计算」，一味一味加配料，看续写范围怎么收窄：

> 🎬 **啥都不加**：续写范围"全网平均"，最可能写成**百科条目式**——又水又泛。（不是模型不行，是范围太宽，只能取平均）
> 🎬 **＋「你是科普专栏作者」**：把分布从全网平均拽向"科普作者"那片区，百科腔、论文腔应声下跌。
> 🎬 **＋「讲给完全没学过物理的高中生」**：多比喻、少术语的文本区被抬高，学术综述几乎出局。
> 🎬 **＋「不出现公式、300 字以内」**：硬锚点，直接封死论文区的路、剪掉长篇百科的尾巴。
> 🎬 **四味齐活**：续写范围收成一条**窄缝**，输出基本"指哪打哪"。

> 小满恍然：「所以提示词里每个字都在给续写'投票'?」
> 元元：「精辟！**字没了，票就没了**——随手撤掉一味，分布立刻弹回去。这就给了你一把**万能检验尺**：以后看到任何'提示词技巧'，先问一句——**它有没有把分布推向我想要的区域？** 讲得通才值得收藏，讲不通多半是时灵时不灵的偏方。」

## 第 2 节　五大技法：招招讲得通，才招招稳定

「这五招覆盖日常九成场景，」元元说，「每招的正例里**没有任何花哨措辞，全是朴素的信息增量**——你是谁、给谁看、长什么样、什么不能做。**改变信息而不是改变语气，这正是工程和玄学的分界线。**」

| 技法 | 反例 → 正例 | 为什么有效（分布语言） |
|---|---|---|
| **① 设角色受众** | "解释量子计算" → "你是科普作者，向高中生解释，多比喻、不用公式" | 角色定"谁在写"、受众定"写给谁"，把分布从最宽最平庸的"百科平均"拽走 |
| **② 给示例 few-shot** | 堆十个形容词 → 给两个满意的范本"照这风格来" | 范例没有歧义，**模式延续是模型最强的本事**;GPT-3 论文最重要发现之一 |
| **③ 给步骤（思维链）** | "直接给答案" → "请一步步思考：先列已知，再写过程，最后给答案" | 多步推理像走钢丝（第15章），让它把中间步骤写出来=**把心算变笔算** |
| **④ 定格式** | "整理成结构化数据" → 给一个 JSON 模板"严格按这个输出" | "结构化"有一万种长法，**模板是上下文里最强的锚点**；交给程序处理时是必需品 |
| **⑤ 划边界** | "我司Q3退货率多少"（它会编） → "资料没提就答'未提供'，不要估算" | 训练数据里"中途承认不会"的下文极少，**给"说不知道"留出路，幻觉率实打实下降** |

> 元元特别点出技法③和⑤：「**'请一步步思考'能让数学题正确率明显上升**——这就是思维链（CoT），第 23 章的推理模型干脆把'先打草稿'训练成了默认动作。而技法⑤是降幻觉的关键：你不开口子，'说不知道'这条路在分布里几乎不存在，模型只能硬编。」

## 第 3 节　system vs user：导演递给演员的人物小传

小满：「我发现有的设定写一遍就一直管用，有的聊几轮就忘了……」

「好观察！这就引出 prompt 的两种角色。」元元打了个戏剧比方，「发给模型的消息分三种：system、user、assistant。这是一出戏——」

> 🎬 **system = 人物小传**（导演开拍前递给演员）
> 放**全局行为约束**：身份口吻、做事准则、安全红线、固定输出格式。先入为主、**压在每一次续写上**，普通用户通常看不见它。

> 🎬 **user = 台词**（开拍后观众递上来的一句句话）
> 放**当前这一件事**：具体问题、材料、临时要求。每轮都在变，跟着对话往前滚。

「为什么全局约束放 system 比塞进 user 更'管用'?」元元揭原理，「两个原因——其一**位置**:system 永远排在对话最前面，整场不动；其二更关键，**训练使然**：第 13 章的对齐阶段，模型被专门调教成'**system 要求优先于 user 要求**'（厂商管这叫指令层级）。」

他演了个对照实验，同一条约束「每次回答不超过 3 句」:

> 🎬 **写进 user 第一句**：第 1 轮乖乖 2 句；聊到第 3 轮，被「逐本介绍 5 本」的强势请求一冲，**直接淹了：9 句**；第 4 轮彻底忘光——约束越聊越往后排，影响力被新内容一轮轮稀释。
> 🎬 **写进 system**：第 1、2、3、4 轮**全程没破功**——位置固定 + 指令层级训练，双保险。

> 元元总结：「ChatGPT 的'自定义指令'、Claude 的项目设定，填的就是 system 这个字段。你见过的各种'人设'——客服、毒舌影评人、苏格拉底导师——本质都是一段你看不见的 system prompt。」

## 第 4 节　长 prompt 结构化：当文档写，别当聊天发

「任务一复杂，prompt 就长到几百上千字，」元元说，「这时最常见的失败不是'模型不会'，而是'**模型没注意到**'——你明明写了'别超过 100 字'，它就当没看见。两条结构原则治这个病：」

> 📐 **原则一 · 信息分区**：背景/任务/约束/示例各立门户，用小标题或分隔线隔开。结构清晰的前文对应训练数据里"说明书、需求文档"那片高质量区域。

> 📐 **原则二 · 重点放两头**：对很长的上下文，模型对**中间部分**利用得最差——研究者称之为 **lost in the middle**（第 17 章拆它的机制）。所以关键约束**开头定调、结尾重申**，别埋在第三段中间等着被看丢。

两条原则合起来，一个像样的长 prompt 长这样：

> **【背景】** 我们是面向程序员的在线教育公司，下周发布《Rust 入门》。
> **【任务】** 写 3 条课程预告文案，发在技术社区。
> **【约束】** 每条 ≤ 60 字；专业克制；不出现「家人们」；不承诺「轻松速成」。
> **【示例】** 上次效果最好的：「指针让你头疼十年？Rust 用所有权换你一夜安眠。」
> **【再次强调】** 只输出 3 条文案本身，不要解释。（把最在乎的放在结尾重申）

> 元元点睛：「看最后一行——这就是'重点放两头'。这份模板眼熟吗？它就是把**五大技法装进了一个有秩序的容器**里。」

## 第 5 节　这些坑，你八成也会踩

**坑一：「'给你 100 美元小费''深呼吸再回答'这类网传咒语是精髓，照抄就灵」**

> ❌ 把个例当规律。
> ✅ 真相是——**能从分布角度讲通的技巧才稳定**；讲不通的偏方时灵时不灵，换个模型、换个版本就失效。

病根：这类偏方多半源于某人某次测试的截图，效应弱、不同模型表现两极。**模型没有账户，只有分布**——"给小费"没增加任何关于任务的信息。五大技法之所以跨模型稳定，是因为它们改变的是**信息本身**（角色、范例、步骤、格式、边界）。拿出那把检验尺：**讲不通分布的，别收藏。**

**坑二：「只要 prompt 写得够好，模型什么问题都能答对」**

> ❌ 把"措辞问题"和"知识问题"混为一谈。
> ✅ 真相是——**prompt 只能引导分布，不能注入模型没有的知识**；知识边界要靠 RAG（第 18 章）和工具调用（第 19 章）来补。

病根：模型不知道你公司上周的会议结论、今天的股价、内部价格表，prompt 再精妙它也只能编。一个实用判断法：**换十种问法都答错或编造的，不是 prompt 的锅**——该去第 18、19 章找答案了。

## 第 6 节　收尾大招：一把检验尺，工程还是玄学

老规矩，秘籍 ＋ 大杀器。

### 提示工程核心，一张表收干净

| 概念 | 一句话 |
|---|---|
| **prompt 本质** | 给文字接龙设续写条件，在高维空间画圈、套性格面具 |
| **五大技法** | 设角色受众 / 给示例 / 给步骤 / 定格式 / 划边界 |
| **system vs user** | 人物小传（全局，压每轮） vs 台词（当前事） |
| **长 prompt** | 信息分区 + 重点放两头（防 lost in the middle） |

### 收尾大招：一把"分布检验尺"，一秒分辨工程与玄学

往后看到任何"提示词神技"，别急着抄，先拿尺子量一句：

> 　🗣️ **「它有没有把分布推向我想要的区域？有没有增加真实的信息？」**
> - 讲得通（角色/范例/步骤/格式/边界）→ **工程，稳定，值得收藏。**
> - 讲不通（给小费/情绪施压/深呼吸）→ **玄学，偏方，时灵时不灵。**
>
> 再配一句知识边界口诀：**换十种问法都治不好的，是知识问题，不是措辞问题**——该上 RAG 或工具了。

### 把整章拧成一句话塞进脑子

> **写提示词不是念咒语，是在模型的高维语义空间里画个圈、套上性格面具，把续写范围从"全网平均"拽到你要的那片知识群落。**
> 五大技法招招改的是"信息"而非"语气"；全局约束放 system（人物小传）比塞 user（台词）更管用。
> 但 prompt 只能引导分布、不能注入新知识——这就是为什么还需要后面的 RAG 和工具。

---

小满把检验尺收好，又冒出个问题：「你刚说'整个对话历史都是续写条件'……那我跟它聊得越久，它要读的东西不就越多？会不会聊着聊着就**记不住开头**、还**越聊越贵**啊？」

元元一拍大腿：「问到下一章的命门了！你这个直觉太准了——这事儿得把第 9 章留下的那张**'平方级握手账单'**重新算一遍。走，下一章我跟你打个**长途电话**聊到第 200 轮，你就懂啥叫'金鱼记忆'和'物理硬上限'了（￣▽￣）」

## 🧰 装进你的工具箱

> **🔑 一句话方法**：写提示词 = 在模型的语义空间里**画个圈、套上性格面具**，把续写范围从"全网平均"拽到你要的群落；五大技法（设角色受众/给示例/给步骤/定格式/划边界）**招招改的是"信息"而非"语气"**；全局约束放 system 比塞 user 更管用。
> **🎯 触发器 · 以后遇到这种情况就掏出它**：看到任何"提示词神技"，拿这把尺子量——**"它有没有把分布推向我想要的区域？有没有增加真实信息？"** 讲得通=工程，讲不通（给小费/深呼吸）=玄学；而**换十种问法都答错的，是知识问题不是措辞问题**（该上 RAG/工具了）。
>
> **✍️ 合上书自测**：
> 1. 为什么"你是儿科医生"这一句能让回答变专业？（用"分布"解释）
> 2. 为什么"请一步步想"能让数学题正确率上升？
> 3. 模型总答不对你公司内部价格表，该继续打磨 prompt 吗？
